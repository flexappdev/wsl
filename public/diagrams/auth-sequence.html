Who can reach /bo.

Every request to a gated route goes through updateSession(). The middleware short-circuits on three checks before it lets the request through, and is env-guarded so a missing Supabase URL never 500s the public site.

The two green steps — forward and 200 OK — are the only path that reaches a /bo page. Everything else terminates at /login. The env-guard at the top exists because an empty Supabase URL throws inside createServerClient and would otherwise 500 every public request too.
